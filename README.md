# Федеративное обучение: FedAvg, FedProx и Adaptive FedProx

Программный модуль для экспериментального исследования федеративного обучения в гетерогенной среде.

## 📋 Описание проекта

Данный проект реализует и сравнивает три алгоритма федеративного обучения:

1. **FedAvg** — базовый алгоритм усреднения весов моделей
2. **FedProx** — оптимизированный алгоритм с проксимальным термом
3. **Adaptive FedProx** ⭐ — **предлагаемый метод** с динамической адаптацией параметра μ

### Элемент научной новизны

**Adaptive FedProx** использует динамическую настройку проксимального параметра μ в зависимости от:
- Номера раунда обучения
- Величины градиентного расхождения между локальными и глобальной моделями

Доступные стратегии адаптации:
- `linear` — линейное уменьшение μ со временем
- `exp` — экспоненциальное уменьшение
- `gradient_based` — адаптация на основе расхождения градиентов

## ⚙️ Конфигурация

Настройки экспериментов задаются в файле `config.yaml`:

```yaml
general:
  seed: 42                    # Seed для воспроизводимости
  rounds: 50                  # Количество раундов обучения
  fraction_fit: 0.3           # Доля клиентов в каждом раунде
  local_epochs: 3             # Количество локальных эпох
  batch_size: 32              # Размер батча
  num_clients: 10             # Количество клиентов
  device: "cuda"              # cuda или cpu

datasets:
  mnist:
    enabled: true
    num_classes: 10
  cifar10:
    enabled: false
    num_classes: 10

heterogeneity:
  alpha: 0.5                  # Параметр Dirichlet (меньше = больше гетерогенность)

fedprox:
  enabled: true
  mu: 0.1                     # Проксимальный параметр

adaptive_fedprox:
  enabled: true
  mu0: 0.5                    # Начальное значение mu
  decay_strategy: "linear"    # linear, exp, gradient_based
  min_mu: 0.01               # Минимальное значение mu

experiments:
  compare_methods: true
  heterogeneity_analysis: true
  alpha_values: [10.0, 1.0, 0.5, 0.1]
  scale_analysis: true
  client_counts: [10, 20, 50, 100]
```

## 🔬 Гетерогенное партиционирование (non-IID)

Используется **распределение Дирихле** для создания non-IID партиций:

```python
from clients.hetero_partitioner import partition_data

client_datasets, statistics = partition_data(
    dataset=train_dataset,
    num_clients=10,
    alpha=0.5,
    num_classes=10,
    seed=42
)
```

Параметр `alpha` контролирует степень гетерогенности:
- **Большой α (10.0)**: данные распределены почти равномерно (IID)
- **Средний α (0.5-1.0)**: умеренная гетерогенность
- **Малый α (0.1)**: сильная гетерогенность (клиенты специализируются на отдельных классах)

## 🧪 Детали реализации

### FedAvg

Стандартное усреднение весов с весами по размеру датасета клиента:

```python
w_global = Σ (n_k / n_total) * w_k
```

### FedProx

Добавляет проксимальный терм к функции потерь:

```python
L_local = L_original + (μ / 2) * ||w_local - w_global||²
```

где `μ` — фиксированный штраф за отклонение.

### Adaptive FedProx ⭐

Динамически изменяет `μ`:

**Linear decay:**
```python
μ(t) = μ₀ * max(0, 1 - t/T_max)
```

**Exponential decay:**
```python
μ(t) = μ₀ * exp(-k * t/T_max)
```

**Gradient-based:**
```python
μ(t) = μ₀ * clip(||Δw|| / max_divergence, 0.1, 1.0)
```

## 📊 Ожидаемые результаты

### Сравнение с централизованным обучением

| Метод | MNIST Accuracy | CIFAR-10 Accuracy |
|-------|----------------|-------------------|
| Централизованное | ~0.98 | ~0.85 |
| FedAvg (α=0.5) | ~0.83 | ~0.65 |
| FedProx (α=0.5) | ~0.89 | ~0.72 |
| **Adaptive FedProx** | **~0.91** | **~0.75** |

### Преимущества Adaptive FedProx

- **+8%** точности по сравнению с FedAvg
- **+2-3%** точности по сравнению с фиксированным FedProx
- **15-20% меньше раундов** до достижения целевой точности
- Лучшая стабильность при экстремальной гетерогенности (α=0.1)
