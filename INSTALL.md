# Инструкция по установке и запуску

## 🚀 Быстрая установка

### Шаг 1: Установка зависимостей

```bash
pip install -r requirements.txt
```

**Примечание:** Для работы с GPU убедитесь, что установлена правильная версия PyTorch для вашей системы. Посетите [pytorch.org](https://pytorch.org) для инструкций.

### Шаг 2: Проверка установки

Запустите файл с примерами:

```bash
python example.py
```

Если команда выполнилась без ошибок, установка прошла успешно.

## 📊 Запуск экспериментов

### Вариант 1: Интерактивное меню (Windows)

```bash
quick_run.bat
```

### Вариант 2: Интерактивное меню (Linux/Mac)

```bash
chmod +x quick_run.sh
./quick_run.sh
```

### Вариант 3: Командная строка

#### Быстрое сравнение методов
```bash
python run_experiments.py --compare --dataset mnist --alpha 0.5
```

#### Полный набор экспериментов
```bash
python run_experiments.py --all
```

#### Анализ гетерогенности
```bash
python run_experiments.py --heterogeneity
```

#### Анализ масштабирования
```bash
python run_experiments.py --scale
```

## ⚙️ Настройка параметров

Отредактируйте файл `config.yaml` для изменения параметров:

```yaml
general:
  rounds: 50              # Количество раундов (измените для быстрых тестов)
  num_clients: 10         # Количество клиентов
  device: "cuda"          # Используйте "cpu" если нет GPU

adaptive_fedprox:
  mu0: 0.5               # Начальное значение mu
  decay_strategy: "linear"  # Стратегия адаптации
```

## 📁 Структура результатов

После запуска эксперимента будут созданы:

- `results/` — CSV и JSON файлы с метриками
- `plots/` — PNG графики и визуализации
- `data/` — скачанные датасеты (MNIST, CIFAR-10)

## 🐛 Устранение проблем

### Ошибка: "No module named 'torch'"

Установите PyTorch:
```bash
pip install torch torchvision
```

### Ошибка: "CUDA out of memory"

Измените `device: "cpu"` в `config.yaml` или уменьшите `batch_size`.

### Ошибка: "ModuleNotFoundError: No module named 'flwr'"

Установите Flower:
```bash
pip install flwr>=1.6.0
```

## 💡 Советы по использованию

1. **Для быстрого тестирования**: уменьшите `rounds` до 10-20 в `config.yaml`
2. **Для GPU**: убедитесь что `device: "cuda"` в конфигурации
3. **Для детального анализа**: используйте `--all` флаг
4. **Для сравнения**: используйте разные значения `alpha` (0.1, 0.5, 1.0, 10.0)

## 📊 Ожидаемое время выполнения

| Эксперимент | CPU | GPU |
|-------------|-----|-----|
| Быстрое сравнение (10 раундов) | ~5-10 мин | ~2-3 мин |
| Полный эксперимент (50 раундов) | ~30-40 мин | ~10-15 мин |
| Все эксперименты | ~2-3 часа | ~40-60 мин |
