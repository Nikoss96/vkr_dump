# 📊 ПОЛНАЯ ИНТЕРПРЕТАЦИЯ РЕЗУЛЬТАТОВ ЭКСПЕРИМЕНТОВ

**Дата анализа:** 25 апреля 2026  
**Эксперименты:** Федеративное обучение на MNIST с различными уровнями гетерогенности  
**Методы:** FedAvg (baseline), FedProx (μ=0.01), Adaptive FedProx  

---

## 🎯 Краткие выводы (TL;DR)

### ✅ **Что работает отлично:**

1. **FedProx при высокой гетерогенности (α=0.1):**
   - ⭐ **Лучший метод:** 98.92% точности
   - 🚀 **В 2.25× быстрее:** 4 раунда vs 9 (FedAvg) до 90%
   - 💚 **Стабильнее:** меньше волатильности

2. **FedAvg при низкой гетерогенности (α≥1.0):**
   - ⭐ **Оптимален:** 99.27-99.32% точности
   - ⚡ **Мгновенная сходимость:** 1 раунд до 90%
   - ✅ **Простота:** не требует настройки μ

### ❌ **Что проблематично:**

1. **Adaptive FedProx:**
   - 💥 **Коллапс на α=0.1:** падение с 98.46% до 90.43% на последнем раунде
   - 🐌 **Медленный старт:** высокий μ₀=0.1 тормозит обучение
   - ⚠️ **Нестабильность:** не рекомендуется для production

## 📈 ГРАФИК 1: Сравнение методов (α=0.1, высокая гетерогенность)

**Файл:** [plots/comparison_alpha0.1_50rounds.png](plots/comparison_alpha0.1_50rounds.png)

### **Ключевые наблюдения:**

#### **Accuracy траектории:**

```
Раунд 0 → Раунд 10 → Раунд 20 → Раунд 50
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
FedAvg:      12% → 86% → 94% → 98.57%
FedProx:     15% → 88% → 96% → 98.92% ⭐
Adaptive:     7% → 87% → 97% → 90.43% 💥
```

#### **Интерпретация:**

**FedProx демонстрирует превосходство:**
- **Быстрый старт:** достигает 90% на раунде 4 (vs 9 у FedAvg)
- **Стабильная траектория:** меньше колебаний (σ=0.015 vs σ=0.032 у FedAvg)
- **Лучший финал:** 98.92% vs 98.57% (+0.35%)

**Проблема Adaptive:**
- Раунды 1-49: нормальная траектория, достигает 98.46%
- **Раунд 50: КАТАСТРОФА** → падение до 90.43%
- Причина: μ(50) ≈ 0.001 (min_mu) → потеря регуляризации

#### **Loss траектории:**

```
FedAvg:      2.30 → 0.20 → 0.10 → 0.056
FedProx:     2.31 → 0.12 → 0.08 → 0.039 ⭐
Adaptive:    2.30 → 0.45 → 0.07 → 0.396 💥
```

**FedProx:** стабильное снижение loss (log scale), финал 0.0385  
**FedAvg:** хорошо, но финал 0.0563 (+46% хуже)  
**Adaptive:** коллапс loss в конце (0.0647 → 0.396, ×6.1)

**Объяснение:**  
При высокой гетерогенности локальные обновления клиентов расходятся (client drift), усреднение менее эффективно.

```
α=10.0: ████████████ 99.30%

**Выводы:**
- ✅ Стабильная производительность на всех α

- 📊 Меньше волатильности: траектории более гладкие

**Объяснение:**  
Проксимальный терм L = L_CE + (μ/2)||w-w_global||² ограничивает расхождение локальных моделей.

#### **3. Adaptive FedProx (правая панель):**

```
α=10.0: ███████████  99.14%
α=1.0:  ██████████   98.92%
α=0.5:  █████████    98.34%

**Выводы:**
- ⚠️ Систематически хуже на всех α
- 💥 Критический коллапс при α=0.1
- 🐌 Медленный старт из-за высокого μ₀

**Объяснение:**  
Линейная стратегия μ(t)=μ₀·(1-t/T) не адаптируется к задаче. В начале μ₀=0.1 слишком велико (тормозит), в конце min_mu=0.001 слишком мало (теряется стабильность).

### **Интерпретация:**

**FedProx — самый стабильный метод:**
- При α=0.1: волатильность в **2.1× ниже** чем FedAvg

### **Раздел 7.2: Ограничения**

#### **Ограничения исследования:**

1. **Датасет:** MNIST слишком простой
   - Даже FedAvg достигает 99%+ при α≥0.5
   - Для сложных задач (CIFAR-10, ImageNet) преимущества могут быть больше

2. **Симуляция:** отсутствие реальных задержек
   - Flower Simulation не моделирует network latency
   - Реальные FL системы имеют отказы клиентов

3. **Adaptive FedProx:** недостаточно исследован
   - Требуется тестирование других стратегий decay (exponential, step-wise)
   - Нужен механизм адаптации к текущему состоянию модели

4. **Статистическая значимость:** один seed
   - Для публикации нужны 3-5 запусков с разными seed
   - Вычислить доверительные интервалы
